## このプロジェクトは何か

過去データで検証して合格(GO)したルールを、コードが毎日機械的に実行する自動売買システム。
Claudeがリアルタイム判断するのではなく、検証済みルールをプログラムが淡々と回す。
一番大事なのは「ダメな戦略を本番前に弾く」こと。

```
fetch_yf.py        … 10年分の株価を取得(分割断崖の自動修復つき) → prices_1306.csv
research_agents.py … 検証してGO/NO-GO判定(IS/OOS+ウォークフォワード) → research_result.json
trading_agents.py  … GO戦略で毎日売買判定(DRY-RUN)→ state.json + history.jsonl
dashboard.html     … スマホで状況確認(GitHub Pages公開済み、心拍確認つき)
```

## これまでの到達点（完了済み）

### 1. データ取得ルートの確立と重大バグ修正
- yfinance(無料10年分)を主軸に、J-Quants(公式・無料2年分)を予備ルートとして確保。
- **重大発見**: yfinanceのデータに1306の1:10分割の断崖(ニセの-90%)が混入していた。
  `fetch_yf.py`に自動検出・修復ロジックを追加済み。データ取得のたびに自動で直る。

### 2. 戦略検証パイプラインの強化
- 基本検証(IS/OOS分割)に加え、**ウォークフォワード検証**(区切りを4パターンずらして再検証)を追加。
  → 1回だけの分割で通った「まぐれのGO」を炙り出す仕組み。
- 損切り幅(5%/8%)も最適化パラメータに組み込み、システムが自動選択するように改良。
- 7種類以上の戦略アーキタイプ(押し目買い/順張り/純RSI逆張り)を試し、純RSI逆張りが生存。

### 3. プロ級の追加検証(記録として3つのラボファイルに保存)
- **bear_lab.py**: 下げ相場シミュレーション。インバースETF常時組込みは10年成績を悪化させると判明
  (+89%→+67%)。200日線トリガー式もV字急落に弱く不採用。→ 現物のみのGO戦略を維持する結論。
- **pro_lab.py**: パラメータ感度マップ(高原の上=まぐれでない)、モンテカルロ1万回
  (マイナス年確率27%、これが心構えの基準値)、運テスト(デタラメ買いの99%に勝利)、
  コスト感度(手数料5倍でも生存)。
- **sizing_lab.py**: 資金500万円想定でのポジションサイジング検証。全額一括/分割エントリー/
  半分だけ運用を比較。分割エントリーは不採用、「半分だけ運用」がリスク調整の最有力候補。

### 4. 運用面の強化(1週間留守にする必要から)
- **取引履歴ログ(history.jsonl)**: 毎日の判定を追記式で記録。同日重複なし(冪等)、
  壊れた行があっても他の記録は守られる仕組み。テスト済み。
- **心拍確認(ダッシュボード)**: state.jsonの更新が止まっていたら赤い警告バナー表示。
  週末を挟んでも誤検知しないよう曜日ごとに許容時間を調整済み(平日30時間、月曜は78時間等)。

### 5. 研究→実行チームの接続、スマホダッシュボード公開
- trading_agents.pyがresearch_result.jsonを読み、検証と同じロジックで判定。GO以外は新規買い停止。
- Git導入→GitHubリポジトリ openfield-studio/tati-bot 作成→push。
- **GitHub Pagesで公開**: https://openfield-studio.github.io/tati-bot/dashboard.html
- state.jsonはまだ.gitignoreで除外中(→スマホでは「サンプル表示」のまま。要対応、下記参照)。

### 6. 製品化(EA販売)の法的調査
- 日本では「投資助言・代理業」の登録要否が販売形態で分かれることを調査済み。
  - 売り切り・パラメータ利用者任せ・販売後の助言なし → 登録不要(GogoJungle等の前例あり)
  - 会員制・継続的シグナル配信・設定への個別アドバイス → 投資助言業の登録が必要
  - 顧客のID/パスワードを預かって代行発注 → さらに重い投資運用業(要登録、資本要件大)
  - 無登録営業は刑事罰(3年以下の懲役or300万円以下の罰金)。
- **本人の意向: 製品化は見送り、個人の資産運用に集中する方針で確定。**
